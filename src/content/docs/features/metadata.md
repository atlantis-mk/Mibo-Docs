---
title: 元数据
description: 了解 Mibo 支持的元数据类型和核心功能特性。
---

元数据让媒体文件从“一个文件名”变成可浏览、可搜索、可识别、可继续观看的媒体条目。Mibo 的
元数据能力围绕影视内容展开，同时为后续更多内容类型保留扩展空间。

## 支持的元数据类型

Mibo 当前重点支持影视类元数据，覆盖常见家庭媒体中心需要展示和管理的信息。

### 电影

电影元数据用于识别单部影片，并补齐用于展示的资料：

- 标题、原始标题、别名。
- 年份、上映日期、地区和语言。
- 剧情简介、标语、评分等展示信息。
- 海报、背景图和其他图片资源。
- 演员、导演、编剧等人物信息。
- 时长、分级、类型和标签。

### 剧集

剧集元数据用于组织系列、季和单集：

- 剧集标题、年份、简介和封面。
- 季信息、季封面和季级别简介。
- 单集标题、集数、播出日期、剧情简介。
- 演员、主创和客串人物。
- 分集级别的海报或剧照。

### 人物

人物元数据用于把演员、导演、编剧等信息从纯文本变成可关联对象：

- 姓名、头像和简介。
- 参与作品和角色名称。
- 与电影、剧集、单集之间的关联。

### 图片资源

图片元数据用于提升浏览体验：

- 海报用于列表、详情页和首页推荐。
- 背景图用于详情页视觉展示。
- 人物头像用于演员和主创展示。
- 分集图用于剧集单集浏览。

### 本地证据

Mibo 也会利用文件名、目录名、层级结构和本地辅助文件作为识别证据。这样即使外部数据不完整，
系统也能从本地结构中提取标题、季集、年份等线索。

## 元数据提供方

Mibo 支持把外部元数据服务配置成可复用的提供方实例。每个实例可以拥有自己的 API Key、基础地址、
语言、超时时间、重试次数和可用状态。

当前支持的元数据提供方包括 TMDB、TVDB 和 MetaTube。提供方实例可以独立启用、停用或进入冷却
状态，方便在限流、认证失败或网络异常时控制影响范围。

### TMDB

[TMDB](https://www.themoviedb.org/) 是常见的电影、剧集、人物和图片数据来源，适合补齐电影、
剧集、演员、海报、背景图、简介、年份等展示资料。Mibo 可以把 TMDB 配置成搜索和详情阶段的默认
提供方，用于大多数影视内容的基础识别与展示。

TMDB 开发者入口：
[TMDB API Documentation](https://developer.themoviedb.org/docs/getting-started)。

### TVDB

[TheTVDB](https://www.thetvdb.com/) 偏向剧集、季、集、播出顺序和电视内容资料，适合需要更精细
剧集结构、分集信息和不同季集顺序的场景。对于剧集收藏较多、分季分集结构复杂的内容，可以把 TVDB
作为搜索或详情补充来源。

TVDB API 入口：
[TheTVDB API](https://www.thetvdb.com/api-information)。

### MetaTube

[MetaTube](https://metatube-community.github.io/) 是面向特定内容生态的元数据项目，提供元数据和
图片提供方能力。Mibo 可以通过 MetaTube 提供方补充 TMDB、TVDB 不覆盖或覆盖不足的内容资料。

MetaTube 项目地址：
[MetaTube GitHub](https://github.com/metatube-community)。

## 元数据模板

元数据模板用于把一组提供方策略保存下来，并应用到不同内容区域。模板可以配置：

- 搜索阶段使用哪些提供方。
- 详情阶段使用哪些提供方。
- 是否允许回退到后续提供方。
- 默认元数据语言。
- 模板描述和适用场景。

例如，可以创建一个 `TMDB 电影模板`，让电影内容优先使用 TMDB 完成搜索和详情补全。具体操作可以
参考 [元数据源配置](/Mibo-Docs/guides/metadata-sources/)。

## 搜索与匹配

元数据搜索负责根据本地文件和目录线索寻找候选条目。Mibo 会综合标题、年份、目录结构、季集信号
等信息，尽量把本地内容匹配到正确的电影、剧集或单集。

匹配结果不只是一次性写入。Mibo 会保留识别过程中的候选、置信度和诊断信息，方便后续复核和治理。

## 详情补全

匹配成功后，Mibo 会继续拉取详情数据，用于补齐简介、海报、背景图、演员、季集结构等展示信息。
详情补全让首页、列表页、搜索页和播放详情页都能使用统一的展示资料。

## 目录结构理解

影视文件经常有不同整理方式：单电影文件、电影文件夹、剧集季目录、合集目录、字幕和花絮混排等。
Mibo 会分析文件名、目录层级和文件类型，判断内容更像电影、剧集、季、单集或辅助资源。

这种理解能力可以减少手工整理成本，也能让复杂目录更容易被系统正确识别。

## 治理与复核

自动识别不可能永远完全正确，因此 Mibo 提供元数据治理能力。管理员可以查看识别结果、待复核项、
异常原因和处理状态，逐步修正错误匹配或补齐缺失资料。

治理能力适合处理：

- 标题相似导致的错误匹配。
- 年份缺失或不准确。
- 剧集季集识别异常。
- 外部提供方没有返回结果。
- 本地目录结构不符合常见规则。

## 多语言与本地化

元数据提供方和模板可以配置默认语言，例如 `zh-CN` 或 `en-US`。这让同一批内容可以优先展示中文
资料，也能在需要时切换到英文或其他语言资料。

## 可扩展方向

Mibo 的元数据能力会继续围绕家庭媒体中心扩展。除了电影和剧集，后续可以承接音乐、照片、直播节目、
自定义集合、人物页和更多外部数据服务，让不同类型内容都拥有一致的识别、展示和治理体验。
