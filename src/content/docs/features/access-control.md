title: 权限控制
description: 使用访问标签和角色规则控制不同用户看到的媒体库内容。
---

Mibo 的权限控制围绕“访问标签”和“角色规则”建立。管理员可以给不同媒体库打上标签，再在角色上配置
允许或拒绝这些标签的规则，从而让不同用户登录后只看到自己应该看到的内容。

这套能力适合家庭成员分级、朋友共享、儿童观看限制、私人内容隔离，以及多人共用同一套服务器但需要
不同展示范围的场景。

## 核心模型

权限控制由三层组成：

- **媒体库访问标签**：给媒体库标记业务含义，例如 `kids`、`family`、`private`、`anime`。
- **媒体库可见模式**：决定媒体库在没有命中角色规则时是默认开放，还是必须命中 allow 规则才展示。
- **角色标签规则**：在角色上配置允许标签和拒绝标签，再把角色分配给用户。

用户登录后，Mibo 会根据用户拥有的角色计算可访问的媒体库集合，并把这个范围用于首页、目录、
详情、资源列表和播放相关接口。

## 访问标签

创建或编辑媒体库时，可以在 **访问标签** 中填写一个或多个标签。标签会被规范化为小写，并按逗号
分隔，例如：

```text
kids, family
```

没有访问标签的媒体库默认对所有已登录用户可见；如果同时启用了“仅对命中 allow 标签规则的角色显示”，
但没有配置任何标签，这个媒体库会处于没有角色能命中的状态，前端也会在媒体库卡片中提示风险。

## 可见模式

每个媒体库支持两种可见模式：

- **默认开放**：没有访问标签时对所有已登录用户可见；有访问标签时，如果用户角色没有配置任何 allow
  标签，也仍然按默认开放处理。
- **仅 allow 角色可见**：只有用户角色命中了该媒体库访问标签的 allow 规则时才展示。

无论哪种模式，只要用户角色命中了该媒体库标签的 deny 规则，就会优先拒绝展示。

## 角色规则

在角色管理中，可以为角色配置两类标签规则：

- **允许标签**：角色可以看到带有这些标签的媒体库。
- **拒绝标签**：角色不能看到带有这些标签的媒体库。

拒绝规则优先级最高。也就是说，如果一个用户同时拥有多个角色，其中一个角色 allow 了 `family`，
另一个角色 deny 了 `family`，最终仍然不能看到带有 `family` 标签的媒体库。

## 展示规则

Mibo 后端的可见性计算逻辑可以概括为：

| 媒体库状态 | 用户角色规则 | 展示结果 |
| --- | --- | --- |
| 没有访问标签，默认开放 | 任意角色 | 可见 |
| 没有访问标签，仅 allow 角色可见 | 任意角色 | 不可见 |
| 有访问标签，命中 deny | 任意 allow 状态 | 不可见 |
| 有访问标签，命中 allow | 未命中 deny | 可见 |
| 有访问标签，默认开放，用户没有任何 allow 标签 | 未命中 deny | 可见 |
| 有访问标签，仅 allow 角色可见，未命中 allow | 未命中 deny | 不可见 |

这个规则让管理员可以先用默认开放快速建立内容，再逐步给敏感内容加标签和角色限制。

## 配置示例

假设服务器中有三个媒体库：

| 媒体库 | 访问标签 | 可见模式 |
| --- | --- | --- |
| 家庭电影 | `family` | 默认开放 |
| 儿童动画 | `kids` | 仅 allow 角色可见 |
| 私人收藏 | `private` | 仅 allow 角色可见 |

可以配置这些角色：

| 角色 | 允许标签 | 拒绝标签 | 效果 |
| --- | --- | --- | --- |
| family | `family` | 空 | 可以看到家庭电影 |
| child | `kids` | `private` | 可以看到儿童动画，看不到私人收藏 |
| owner | `family, kids, private` | 空 | 可以看到全部受控内容 |

把对应角色分配给用户后，用户首页和浏览入口会自动按规则收敛，只展示允许访问的媒体库内容。

## 代码中的行为

Mibo 后端把媒体库标签保存为 `library_access` 类型标签，并通过媒体库与标签的关联表记录归属。
角色上的 allow 和 deny 规则会保存为角色与标签规则。

用户请求需要内容可见性的接口时，后端会：

- 读取当前用户的角色。
- 汇总这些角色的 allow 和 deny 标签规则。
- 遍历媒体库及其访问标签，计算当前用户可访问的媒体库 ID 列表。
- 把可访问范围写入请求上下文。
- 由目录、首页、详情、资源和播放接口使用这个上下文过滤结果。

因此，权限不是只在前端隐藏按钮；后端接口也会根据当前用户重新计算访问范围。用户请求指定媒体库时，
如果该媒体库不在可访问范围内，会返回禁止访问。

## 全局回滚开关

系统设置中有 **关闭媒体库可见性强制校验** 开关。它只适合兼容旧数据或紧急回滚使用。开启后，
后端会按默认开放方式返回媒体库范围，但已经配置好的标签和角色规则不会被删除。

正常使用时建议保持这个开关关闭，让访问标签和角色规则持续生效。

## 使用建议

建议先按内容边界设计少量稳定标签，例如 `kids`、`family`、`private`，再把标签复用到多个媒体库。
标签过多会增加维护成本，也容易让角色规则变得难以审计。

对于需要严格隔离的内容，建议同时设置访问标签，并开启“仅对命中 allow 标签规则的角色显示”。对于
普通共享内容，可以保持默认开放，只在需要排除某些角色时使用 deny 规则。
