Mibo 的播放能力同时覆盖站内播放和外部播放器调用。默认情况下，用户点击 **播放** 会进入 Mibo
内置播放器；如果更习惯使用 VLC、IINA、mpv 等桌面或移动端播放器，也可以把播放链接直接交给外部
播放器打开。

## 本地播放器

Mibo 内置播放器用于提供完整的站内播放体验。它适合希望在浏览器里完成观看、恢复进度、字幕切换和
播放控制的用户。

内置播放器支持：

- **播放进度恢复**：根据用户观看记录自动定位到上次观看位置。
- **自动播放下一集**：剧集播放结束后可以自动进入下一集。
- **优先直放**：设备支持时优先使用原始媒体文件，减少不必要的转换。
- **字幕偏好**：可以配置默认字幕模式，并在播放中切换字幕。
- **播放模式切换**：播放页可以在可用播放模式之间切换。
- **外部播放器入口**：即使当前进入了内置播放器，也可以从播放页菜单选择“用外部播放器打开”。
- **片头片尾设置**：播放页提供片头片尾秒数设置，便于跳过固定片段。
- **资源探测**：对可探测资源刷新音轨、字幕等媒体信息。

内置播放器的优势是和 Mibo 的账号、进度、字幕、播放页控制保持一致，适合大多数 Web、平板和电视
浏览器场景。

## 外部播放器调用

外部播放器模式会先向 Mibo 请求当前内容的播放地址，再把这个地址转换成播放器支持的 URL Scheme
或 Intent，然后交给本机安装的播放器处理。

当前支持的外部播放器包括：

- IINA
- PotPlayer
- VLC
- Android Intent
- nPlayer
- OmniPlayer
- Fig Player
- Infuse
- Fileball
- MX Player
- MX Player Pro
- iPlay
- mpv

Mibo 会根据当前浏览器平台筛选适合的播放器。例如 macOS 下会优先展示 IINA、VLC、OmniPlayer、
Infuse、Fileball、mpv 等选项；Windows、Android、iOS 和 Linux 会展示各自可用的播放器入口。

外部播放器适合这些场景：

- 希望使用本机播放器的解码能力和快捷键。
- 需要播放器自己的字幕、音轨、画质、滤镜或倍速能力。
- 浏览器对当前格式兼容性不足。
- 手机或平板上希望直接拉起已安装的视频播放器。

需要注意的是，外部播放器当前会直接打开播放链接，不会自动同步站内的恢复进度和字幕控制。需要完整
站内体验时，建议使用 Mibo 内置播放器。

## 全局播放打开方式

可以在 **设置 > 外观与布局 > 播放** 中配置点击 **播放** 后的默认行为：

![播放设置页面](/Mibo-Docs/screenshots/features/display-player-settings.png)

这里可以配置：

- **自动播放下一集**：控制剧集播放结束后是否自动继续。
- **优先直放**：控制是否优先使用原始媒体文件。
- **默认字幕模式**：设置自动、始终显示或默认不显示字幕。
- **播放打开方式**：选择默认进入 Mibo 内置播放器，或直接调用外部播放器。
- **外部播放器**：当播放打开方式选择外部播放器时，指定默认拉起的播放器。

播放打开方式属于当前浏览器的偏好设置。也就是说，同一个账号在不同设备或不同浏览器中，可以分别
选择适合当前设备的播放方式：客厅电视可以使用内置播放器，Mac 可以默认调用 IINA，手机可以默认
调用系统支持的移动端播放器。

## 使用建议

如果你希望 Mibo 负责完整的观看体验，包括进度、字幕和播放页控制，推荐使用 **Mibo 内置播放器**。

如果你更重视本机播放器的格式兼容、硬件解码、字幕渲染或操作习惯，可以把全局播放打开方式切换为
**外部播放器**，再选择当前设备上实际安装的播放器。

对于不同成员或不同设备，可以按设备分别配置：家庭公共设备使用内置播放器保持一致体验，个人电脑或
移动设备使用外部播放器获得更强的本地播放能力。
