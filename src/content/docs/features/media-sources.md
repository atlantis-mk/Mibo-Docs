Mibo 的媒体源能力用于把不同位置的内容接入同一个家庭媒体系统。你可以直接读取服务器上的本地
目录，也可以通过 OpenList 把网盘、远程目录和其他文件服务统一成一个可浏览的网络地址。

## 多来源接入

Mibo 支持两类核心接入方式：

- **本地路径**：适合硬盘、NAS 挂载目录、Docker 映射目录，以及服务器可以直接读取的文件路径。
- **OpenList 网络地址**：适合把多个网盘或远程存储先挂载到 OpenList，再让 Mibo 通过
  OpenList 地址访问这些内容。

这种组合让部署方式更灵活：常用内容可以放在本地路径里，网盘内容可以交给 OpenList 聚合，Mibo
只需要面对统一的目录浏览和文件访问能力。

## 本地路径

本地路径是最直接的接入方式。只要运行 Mibo 的系统账号能读取该目录，就可以把它添加进来。

适合本地路径的场景包括：

- 服务器内置硬盘或外接硬盘。
- NAS 通过 SMB、NFS、WebDAV 等方式挂载到本机后的目录。
- Docker、Compose 或系统服务中映射给 Mibo 的媒体目录。
- 已经由其他同步工具下载到本机的文件夹。

本地路径的优势是结构简单、延迟低、权限边界清楚。对于长期收藏、高频播放或需要稳定扫描的内容，
优先使用本地路径通常更稳。

## OpenList 网络地址

OpenList 是一个可以聚合多种存储驱动的文件列表服务。它把不同网盘或远程存储挂载到自己的目录树中，
再通过一个网络地址对外提供浏览、下载、WebDAV 或代理访问能力。

在 Mibo 中使用 OpenList 时，你只需要填写 OpenList 的访问地址、用户名和密码，然后选择 OpenList
中已经挂载好的目录。详细操作可以参考：
[OpenList 使用方法](/Mibo-Docs/guides/openlist/)。

OpenList 官方通用驱动文档：
[Common - OpenList Docs](https://doc.oplist.org/guide/drivers/common)。

## 可接入的来源

Mibo 原生接入本地路径；通过 OpenList，则可以间接接入 OpenList 支持的网盘和远程存储。

根据 OpenList 的通用驱动文档，OpenList 的通用挂载配置包含挂载路径、排序、备注、引用认证、
签名、索引、缓存、Web 代理、WebDAV 策略、下载代理和排序等能力。其中，文档明确提到可复用认证
引用的网盘包括：

- 中国移动云盘（139Yun）
- 阿里云盘 Open（AliyundriveOpen）
- 天翼云盘客户端（189CloudPC）
- 123 云盘分享（123PanShare，引用 123Pan）
- Cloudreve V3 / V4

OpenList 还提供独立驱动页面和可扩展驱动机制，因此实际可用来源会随 OpenList 版本和你安装的驱动
变化。配置前建议以 OpenList 官方文档和后台驱动列表为准。

## OpenList 的特性

OpenList 不只是“把网盘列出来”。对 Mibo 来说，它更像一个远程内容网关，负责把不同存储的差异
收敛成统一目录。

### 统一挂载路径

OpenList 使用挂载路径组织不同驱动。每个挂载项都有一个对外展示的位置，例如 `/电影`、
`/剧集`、`/阿里云盘`。Mibo 连接 OpenList 后，可以像浏览普通目录一样选择这些路径。

### 多网盘聚合

OpenList 可以把多个网盘或远程服务放进同一个目录树里。你可以把不同来源按用途整理，例如电影、
剧集、纪录片、家庭视频分别来自不同网盘，但在 Mibo 里仍然按统一入口使用。

### 认证复用

OpenList common 文档说明，部分网盘可以通过 `ref:/挂载路径` 的方式从已挂载存储引用认证、令牌
等信息，减少重复配置同一账号或 token 的成本。

### 目录缓存

OpenList 支持目录结构缓存和自定义缓存策略。对于网盘目录，缓存可以减少频繁请求，提高浏览体验；
对于更新较快的目录，也可以单独设置更短的缓存时间。

### Web 代理与 WebDAV 策略

OpenList 可以控制网页预览、下载、直链和 WebDAV 访问是否走代理。常见策略包括 302 跳转到真实
链接、使用代理 URL，或通过本机代理中转。不同网盘的限制不同，使用时需要根据账号风险、网络质量
和播放兼容性选择。

### 下载代理

OpenList 支持配置下载代理 URL，也可以配合 Cloudflare Workers、通用二进制代理或自建代理程序。
这让远程播放和跨网络访问有更多部署选择。

### 签名与防扫

OpenList 支持对指定驱动启用签名，让文件访问带有签名保护。对于不希望被随意扫描或公开访问的
路径，可以通过签名和代理策略增加一层保护。

### 排序与文件夹展示

OpenList 提供排序方向、排序字段，以及文件夹置前或置后的配置。目录结构比较复杂时，这些设置可以
帮助 Mibo 用户在路径选择时更容易找到目标内容。

## 使用建议

如果内容已经稳定保存在服务器或 NAS 上，优先使用本地路径；如果内容分散在多个网盘、远程服务或
朋友共享的 OpenList 上，可以先用 OpenList 聚合，再把 OpenList 地址接入 Mibo。

对于大体积视频，建议在正式使用前测试目录浏览、播放直链、代理策略和账号限制。网盘播放体验通常
取决于 OpenList 所在机器、网盘服务质量、网络带宽、代理方式和账号策略。
